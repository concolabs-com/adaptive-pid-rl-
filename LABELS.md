# Recommended GitHub Labels

## Type Labels
```txt
feature
bug
documentation
chore
refactor
security
test
ci
```

## Priority Labels
```txt
priority-low
priority-medium
priority-high
priority-critical
```

## Status Labels
```txt
status-blocked
status-in-review
status-ready
status-in-progress
status-needs-info
```

## Area Labels
```txt
frontend
backend
database
api
auth
ui
deployment
```

## Recommended Label Rules
- Every issue should have at least one type label.
- Every urgent issue should have a priority label.
- Use area labels only when they add clarity.
- Avoid creating too many labels.
